- **Chunking Sub-package**: Implements a three-tier strategy (`smart_chunk`) routing DOCX to heading-based splitting and PPTX/PDF to section-based splitting, with sentence-boundary fallback for oversized segments (`academic_chunker.py`).
- **Retrieval Sub-package**: Provides a `CrossEncoderReranker` that blends retrieval scores with cross-encoder logits (using `BAAI/bge-reranker-v2-m3` or `ms-marco-MiniLM-L-6-v2`) and includes heuristic fallbacks (`reranker.py`). The `enhanced_reranker.py` file exists but contains severe syntax corruption and is excluded from the active API (`__init__.py`).
- **Verification Sub-package**: Offers local, API-free validation via `CitationVerifier` (token overlap checks) and `compute_confidence` (weighted aggregation of retrieval, reranking, and coverage metrics) (`citation_verifier.py`, `confidence_score.py`).
- **Interface Boundaries**: Each sub-package exposes a functional API via `__init__.py`, relying on `langchain_core.documents.Document` as the shared data structure.
